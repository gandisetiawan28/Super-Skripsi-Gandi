# Implementation Plan: Smart Licensing & Auto-Update System

Sistem lisensi dan pembaruan otomatis (auto-update) menggunakan Google Sheets sebagai database, Apps Script sebagai API, dan Vercel/Google Drive sebagai hosting file.

## Arsitektur Sistem

1. **Licensing & Config Center (Google Sheets)**: 
   - Sheet `Licenses`: Data lisensi dan perangkat.
   - Sheet `AppConfig`: Data versi terbaru, link download, dan flag force update.
2. **API Gateway (Apps Script)**: Endpoint tunggal untuk validasi lisensi dan pengecekan versi.
3. **Hosting & Deployment**:
   - **Word Add-in**: Hosted di Vercel (Auto-deploy dari private GitHub).
   - **Flutter App/Extension**: Hosted di Google Drive (Publik via link).

## Rencana Perubahan

### 1. Database Schema (Google Sheets)
Sheet baru `AppConfig`:
- `platform`: Windows, Android, atau Extension.
- `latest_version`: String versi (misal: "1.0.5").
- `download_url`: Link download langsung dari Google Drive.
- `force_update`: Boolean (TRUE/FALSE) untuk memaksa user update.

### 2. Service & Logic Update
#### [NEW] `lib/services/update_service.dart`
- Menggunakan `package_info_plus` untuk cek versi lokal.
- Bertanya ke Apps Script tentang `latest_version`.
- Jika `local_version < latest_version`, tampilkan dialog pembaruan.

#### [MODIFY] `lib/services/license_validation_service.dart`
- Menambahkan fungsi `checkUpdate()` yang dipanggil saat aplikasi startup.

### 3. Hosting Strategy
- **Vercel**: Menghubungkan repositori private GitHub untuk Word Add-in (React).
- **Google Drive**: Menyimpan file `.exe`, `.apk`, dan `.zip` (ekstensi) dengan akses "Anyone with link".

## User Review Required

> [!IMPORTANT]
> **Package Info**: Kita perlu menambahkan dependensi `package_info_plus` ke `pubspec.yaml` untuk membaca versi aplikasi secara otomatis.

> [!TIP]
> **Force Update**: Jika Anda menemukan bug kritikal, Anda bisa mengubah `force_update` menjadi `TRUE` di Google Sheets, sehingga user tidak bisa menggunakan aplikasi sebelum mereka mengunduh versi terbaru.
